# Requirements Specification: Codex Installation and Doctor Paths

**Item**: REQ-0115 | **GitHub**: #179 | **CODEX**: CODEX-046 | **Phase**: 8 | **Workstream**: C
**Status**: Analyzed | **Depends on**: REQ-0089, REQ-0114

---

## 1. Business Context

The provider-aware installer (REQ-0089) dispatches to provider-specific install/update/uninstall/doctor functions. This item implements the Codex-side functions. The Codex installer is simpler than the Claude installer (no hooks directory, no settings.json merge, no MCP config) but follows the same API shape so the dispatcher can treat all providers uniformly.

## 2. Functional Requirements

### FR-001: Install
- **AC-001-01**: `installCodex(projectRoot, options)` creates the Codex instruction directory (`.codex/` or equivalent).
- **AC-001-02**: Copies core config into the instruction directory.
- **AC-001-03**: Generates Codex-specific instruction files from team specs and team instances.
- **AC-001-04**: Returns `{ success: boolean, filesCreated: string[], errors: string[] }`.

### FR-002: Update
- **AC-002-01**: `updateCodex(projectRoot, options)` regenerates Codex instruction files from latest core models.
- **AC-002-02**: Preserves user-modified content (detects user edits via content hash comparison).
- **AC-002-03**: Returns `{ success: boolean, filesUpdated: string[], filesSkipped: string[], errors: string[] }`.

### FR-003: Uninstall
- **AC-003-01**: `uninstallCodex(projectRoot, options)` removes Codex-specific generated files.
- **AC-003-02**: Preserves user-created content that was not generated by the installer.
- **AC-003-03**: Returns `{ success: boolean, filesRemoved: string[], filesPreserved: string[], errors: string[] }`.

### FR-004: Doctor
- **AC-004-01**: `doctorCodex(projectRoot)` validates the Codex installation.
- **AC-004-02**: Checks: instruction files exist, core config present, team specs loadable.
- **AC-004-03**: Returns `{ healthy: boolean, checks: [{ name: string, passed: boolean, message: string }] }`.

### FR-005: API Parity
- **AC-005-01**: All 4 functions mirror the Claude installer API signature (same parameter shape, same return shape).
- **AC-005-02**: The provider-aware installer (REQ-0089) can call Codex functions identically to Claude functions.

## 3. Out of Scope

- Codex CLI binary management (install, version check, updates)
- Modifying the Claude installer
- Runtime Codex task execution

## 4. MoSCoW

FR-001, FR-002, FR-003, FR-004, FR-005: **Must Have**.
